# lcmm-ws

`lcmm-ws` это небольшая библиотека для LCMM-приложений, которым нужен
практичный и внятный каркас для работы с веб-сокетом, без попытки превратить
весь стек в громоздкую систему на все случаи жизни.

Библиотека закрывает:

1. хранение websocket-сессий;
2. подписки по темам;
3. рассылку сообщений подписчикам;
4. базовые служебные сообщения для браузера;
5. адаптер для `http-kit`;
6. разбор типовых сообщений первой фазы: `ping`, `subscribe`, `unsubscribe`;
7. типовую обработку подписки и отписки;
8. `ws-registry`, через который модули могут в `init!` регистрировать свой
   websocket-экспорт.

Библиотека сознательно не закрывает:

1. правила открытия соединения;
2. проверку `Origin`;
3. проверку входа и прав доступа на стороне приложения;
4. распределенное хранение сессий;
5. гарантированную доставку после разрыва связи;
6. общий каркас для произвольных клиентских команд.

## Основная идея

Веб-сокет в LCMM остается транспортным слоем приложения.

Приложение владеет:

1. `/ws`;
2. проверкой открытия соединения;
3. `Origin`;
4. проверкой входа;
5. лимитами и жизненным циклом соединения;
6. общей интеграцией с `lcmm-guard`.

Модули владеют:

1. смыслом своих событий;
2. правилами видимости подписок;
3. переводом внутреннего события модуля в сообщение для браузера.

Чтобы при добавлении нового модуля не приходилось заново писать
транспортную склейку,
модуль может регистрировать свои websocket-возможности через `ws-registry`.

## Пространства имен

1. `lcmm-ws.transport` - минимальный контракт транспортного слоя.
2. `lcmm-ws.core` - хранилище сессий, подписки и рассылка.
3. `lcmm-ws.limits` - простые лимиты по сообщениям и подпискам.
4. `lcmm-ws.codec` - базовые служебные сообщения для браузера.
5. `lcmm-ws.registry` - реестр websocket-экспорта модулей.
6. `lcmm-ws.bridge` - переход от внутреннего события к доставке по теме.
7. `lcmm-ws.protocol` - разбор типовых входящих сообщений и нормализация тем.
8. `lcmm-ws.flow` - типовая обработка подписки и отписки.
9. `lcmm-ws.http-kit` - адаптер для `http-kit`.

## Что добавилось во второй итерации

Во второй итерации библиотека получила еще один узкий слой удобства:

1. `lcmm-ws.protocol/parse-client-message` разбирает сообщения первой фазы;
2. `lcmm-ws.protocol/wire-topic->internal` и `internal-topic->wire` переводят
   тему между браузерной и внутренней формой;
3. `lcmm-ws.flow/process-subscribe!` убирает повторяющийся ход
   `найти обработчик -> проверить доступ -> проверить лимит -> записать подписку`;
4. `lcmm-ws.flow/process-unsubscribe!` так же убирает повторяющийся код
   отписки.

Важно:

это не общий язык клиентских команд, а только узкий и намеренно ограниченный
слой для первой фазы websocket-протокола.
